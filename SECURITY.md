# Security Policy

## Reporting a Vulnerability

- Email the maintainers at `security@example.com` with details and reproduction steps (replace with your project contact before release).
- Do not open public issues for security vulnerabilities.
- Expect an acknowledgment within 5 business days.

## Supported Versions

- Main branch and the latest stable release are supported with security fixes.

## Dependency Policy

- Dependencies are kept up-to-date via Dependabot (weekly).
- Security updates are prioritized; CI must remain green after upgrades.
